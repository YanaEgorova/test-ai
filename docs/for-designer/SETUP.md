# SETUP.md — первая настройка

Один раз. Минут 30. Дальше эти штуки уже не трогаешь.

После этой инструкции ты сможешь: открыть проект, запустить его у себя на компьютере (увидишь сайт в браузере), сделать первый компонент через Claude, и отправить его разработчику на ревью.

## Что нужно установить

Три приложения. Все бесплатные.

### 1. Claude desktop

Скачай с [claude.ai/download](https://claude.ai/download). Запусти, войди через свой email от Anthropic.

**Важно:** проверь, что у тебя доступен **режим Cowork** (это режим работы Claude с папкой на компьютере). Если в Claude desktop нет этой функции — напиши разработчику, может потребоваться запрос доступа (Cowork пока в исследовательском режиме).

### 2. GitHub Desktop

Скачай с [desktop.github.com](https://desktop.github.com). Это приложение для работы с проектом через кнопки, без терминала.

Запусти, войди через свой GitHub-аккаунт. Если аккаунта нет — зарегистрируйся на [github.com](https://github.com), это две минуты.

После регистрации **попроси разработчика добавить тебя в проект как collaborator** — он пришлёт приглашение на твою почту, открой и нажми "Accept".

### 3. Node.js

Скачай LTS-версию с [nodejs.org](https://nodejs.org). Установи как обычное приложение — один клик "Next" во всех окошках.

Node нужен, чтобы запускать проект у себя локально и видеть сайт в браузере. Сам с ним работать не будешь — он просто стоит в фоне.

## Скачай проект

В GitHub Desktop:

1. Меню **File → Clone a Repository**
2. Во вкладке "GitHub.com" найди проект `last-landing` (он появится после того как разработчик пригласил тебя)
3. **Local Path** — выбери куда сохранить (рекомендую `~/Documents/last-landing`)
4. Нажми **Clone**

Готово, проект у тебя на компьютере.

## Запусти проект (первый раз)

После клонирования в GitHub Desktop сверху есть меню **Repository**:

1. **Repository → Open in Terminal** (или горячая клавиша Cmd+T) — откроется чёрное окно (терминал) в папке проекта.
2. В терминале набери и нажми Enter:

   ```
   npm install
   ```

   Подожди 1–3 минуты. Будут бежать строчки — это нормально. Дождись, пока курсор снова станет активным.

3. Дальше набери:

   ```
   npm run dev
   ```

   Появится сообщение типа `Local: http://localhost:3000`. Не закрывай терминал.

4. Открой в браузере [http://localhost:3000](http://localhost:3000) — увидишь демо-страницу с заголовком, типографикой и карточкой отзыва. Это значит всё работает.

В следующий раз когда сядешь работать — только пункт 1 и 3 (`npm install` нужен один раз, `npm run dev` — каждый раз).

## Подключи Cowork

1. Открой Claude desktop.
2. Открой режим Cowork.
3. Выбери папку проекта (`~/Documents/last-landing`, если ты её туда сохранила).
4. В новом чате напиши:

   > _Я дизайнер, делаю компоненты для сайта Last landing. Следуй CLAUDE.md._

5. Claude подтвердит что готов — и можешь начинать.

## Сделай первый компонент (для практики)

1. В чате напиши что-то вроде:

   > _Сделай простую карточку с тремя кнопками: "Купить", "В избранное", "Поделиться". На карточке заголовок "Тестовая карточка" и абзац текста._

   Скрин Figma не нужен — это просто практика.

2. Claude создаст файл и расскажет где он лежит.
3. Открой браузер с `http://localhost:3000` — обнови страницу (или подожди секунду). Если использовать карточку на главной — попроси Claude добавить её туда временно для проверки.

Если видишь карточку на странице — всё работает.

## Отправь на ревью

В GitHub Desktop:

1. Слева увидишь "1 changed file" — это твой новый компонент.
2. Сверху текущая ветка пишется как **Current Branch: main**. Нажми на неё → **New Branch**.
3. Введи имя ветки (на английском, без пробелов): например `add-test-card`.
4. Нажми **Create Branch**.
5. Внизу слева — поле "Summary". Напиши коротко что сделала: _"add test card"_.
6. Нажми синюю кнопку **Commit to add-test-card**.
7. Сверху появится кнопка **Publish branch** — нажми.
8. После публикации появится **Create Pull Request** — нажми, откроется GitHub в браузере.
9. На GitHub проверь что заголовок понятный, при желании добавь описание ("тестовая карточка для проверки потока"), нажми **Create pull request**.

Разработчик получит уведомление и посмотрит твой компонент. Дальше:

- Если всё ок — он смерджит и компонент попадёт в основной проект.
- Если что-то не так — оставит замечание прямо в PR. Открой Claude, скопируй замечание, попроси переделать. После правок снова **Commit** + Push (publish) в **ту же ветку** — твой PR обновится автоматически.

## Каждый день

Когда садишься работать:

1. Открой GitHub Desktop → нажми **Fetch origin** сверху, потом если есть изменения — **Pull origin**. Это подтянет изменения, которые разработчик мог сделать.
2. Открой терминал (Cmd+T в GitHub Desktop) → `npm run dev` (если ещё не запущен).
3. Открой Claude desktop, выбери папку проекта в Cowork.
4. Работай.

В конце дня (или после каждой задачи) — Commit + Publish + Pull Request как выше.

## Чего НЕ делай

- **Не редактируй файлы в коде вручную** — у тебя нет редактора кода, и это правильно. Если что-то не так — попроси Claude поправить.
- **Не запускай в терминале команды, которые сама не понимаешь.** Из всех команд тебе нужны только две: `npm install` и `npm run dev`. Всё остальное — через GUI приложений.
- **Не работай напрямую в ветке `main`.** Всегда создавай новую ветку (см. шаг 2 в "Отправь на ревью"). Это страховка: если что-то пойдёт не так, можно отменить — основной проект не пострадает.

## Если что-то не работает

| Симптом                                 | Что делать                                                                                                                       |
| --------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `npm: command not found`                | Node.js не установлен или не подцепился — перезагрузи компьютер после установки                                                  |
| `http://localhost:3000` не открывается  | В терминале не запущен `npm run dev`. Открой Terminal в GitHub Desktop и запусти.                                                |
| В Claude desktop нет Cowork             | Cowork в исследовательском режиме, может потребоваться доступ — напиши разработчику                                              |
| GitHub Desktop не видит репо            | Разработчик ещё не пригласил тебя в проект, проверь email или попроси прислать заново                                            |
| Что-то сломалось в коде, не запускается | **Не чини сама.** Скриншот ошибки → разработчику. Откатить ветку можно: в GitHub Desktop меню Repository → "Discard all changes" |
| Claude генерирует не то                 | Скрин + 1-2 конкретных замечания. Не пиши "сделай красивее" — пиши "карточка должна быть шире, кнопка справа"                    |
