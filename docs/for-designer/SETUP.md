# SETUP.md — первая настройка

Один раз. Минут 20. Дальше — работаешь только через Claude и двойной клик.

После этой настройки твой повседневный поток будет таким:

1. Двойной клик `start.command` → сайт запущен.
2. Открыла Claude → описала компонент → Claude его сделал, ты увидела в браузере.
3. Сказала Claude _"сохрани и отправь на ревью"_ → готово.

Ни терминала, ни git'а, ни кнопок GitHub Desktop. Всё в одном окне Claude.

## Что нужно установить

Три приложения, все бесплатные.

### 1. Claude desktop

Скачай с [claude.ai/download](https://claude.ai/download). Запусти, войди через свой email.

Проверь, что у тебя доступен **режим Cowork** (это режим, в котором Claude может работать с папкой проекта на твоём компьютере). Если в Claude нет этой функции — напиши разработчику, может потребоваться запрос доступа (Cowork пока в исследовательском режиме).

### 2. Node.js

Скачай LTS-версию с [nodejs.org](https://nodejs.org). Установи как обычное приложение — один клик "Next" во всех окошках.

Node нужен чтобы запускать сайт у себя на компьютере. Сама с ним работать не будешь — он просто стоит в фоне.

### 3. GitHub CLI

Скачай с [cli.github.com](https://cli.github.com) — на Mac есть готовый установщик `.pkg`. Установи.

Это маленькая утилита, чтобы Claude мог отправлять твои изменения на GitHub без долгой настройки. Запускать её сама ты будешь **один раз** для входа в аккаунт — дальше Claude использует её под капотом.

## Первый вход в GitHub

Зарегистрируйся на [github.com](https://github.com), если ещё нет аккаунта (минута).

Открой **Terminal** (на Mac — встроенное приложение, иконка чёрный квадрат с символом `>_`. Найди через Spotlight: Cmd+Space → "Terminal" → Enter).

В терминале набери одну команду и нажми Enter:

```
gh auth login
```

Дальше следуй подсказкам:

- "What account do you want to log into?" → выбери **GitHub.com** (стрелками + Enter).
- "What is your preferred protocol?" → **HTTPS**.
- "How would you like to authenticate?" → **Login with a web browser**.
- Появится 8-значный код вида `XXXX-XXXX` и предложение нажать Enter — нажми, откроется браузер.
- В браузере введи этот код, разреши доступ.

Готово, gh запомнит твою авторизацию навсегда. Теперь Claude сможет работать с GitHub от твоего имени.

## Получи проект

Когда разработчик пригласит тебя в проект (придёт письмо на email — открой и нажми "Accept"), скопируй проект к себе.

В том же терминале одной командой:

```
gh repo clone <owner>/last-landing ~/Documents/last-landing
```

(точный адрес `<owner>/last-landing` пришлёт разработчик)

После этой команды проект окажется в папке `~/Documents/last-landing`.

Терминал можно закрыть. Больше он тебе не понадобится для работы с проектом.

## Подключи Cowork

1. Открой Claude desktop.
2. Перейди в режим Cowork.
3. Выбери папку `~/Documents/last-landing`.
4. В новом чате напиши:

   > _Я дизайнер, делаю компоненты для сайта Last landing. Следуй CLAUDE.md._

5. Claude подтвердит — готов работать.

## Запусти сайт

В папке проекта (открой через Finder: Cmd+Space → "last-landing" → Enter) найди файл **`start.command`** (на Windows — `start.bat`).

Дважды кликни. Откроется чёрное окно Terminal, побегут строчки, в конце появится:

```
Ready in 1.2s
Local:        http://localhost:3000
```

Открой эту ссылку в браузере — увидишь демо-страницу сайта.

**Не закрывай это окно Terminal.** Пока оно открыто — сайт работает, и ты будешь видеть свои изменения в браузере мгновенно. Закроется — сайт остановится, нужно будет запустить снова.

> ⚠️ В первый раз Mac может ругаться "файл из ненадёжного источника". Тогда: правый клик на `start.command` → "Открыть" → в диалоге безопасности тоже "Открыть". После этого всегда можно двойным кликом.

## Готова работать

Открой Claude → опиши компонент простыми словами → приложи скриншот. Claude создаст файл, ты увидишь результат в браузере. Когда готово — попросишь Claude отправить на ревью.

## Команды для Claude (в чате)

Это вся git-механика, которая тебе нужна. Просто напиши Claude одну из этих фраз:

| Что пишешь                      | Что Claude сделает                                             |
| ------------------------------- | -------------------------------------------------------------- |
| _"подтяни последние изменения"_ | Обновит проект свежими правками от разработчика                |
| _"покажи что я изменила"_       | Расскажет простыми словами какие файлы поменялись              |
| _"сохрани и отправь на ревью"_  | Создаст ветку, отправит изменения на GitHub, даст ссылку на PR |
| _"отмени всё что я сделала"_    | Вернёт файлы как было (с подтверждением, не сразу)             |

Перед коммитом и отправкой Claude **всегда показывает** что собирается отправить — ты подтверждаешь, и только тогда он отправляет. Случайно ничего не уйдёт.

## Каждый день — короткая версия

1. Двойной клик `start.command` → сайт запущен.
2. Открываешь Claude desktop, проект уже выбран в Cowork.
3. Первое сообщение в новом чате: _"Я дизайнер, подтяни последние изменения"_.
4. Работаешь — описываешь компоненты, прикладываешь скрины, проверяешь в браузере.
5. Когда закончила задачу: _"сохрани и отправь на ревью"_ → Claude всё сделает.

## Чего НЕ делай

- **Не правь файлы в коде вручную** — у тебя нет редактора кода, и это правильно. Если что-то не так — попроси Claude поправить.
- **Не запускай в терминале команды, которые сама не понимаешь.** Из всех команд тебе нужны только три, и все они одноразовые при первой настройке: `gh auth login`, `gh repo clone ...`. Дальше — только Claude и двойной клик.
- **Не закрывай окно Terminal от `start.command` во время работы** — оно держит сайт запущенным.

## Если что-то не работает

| Симптом                                                  | Что делать                                                                                                                      |
| -------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| `npm: command not found`                                 | Node.js не подцепился — перезагрузи компьютер после установки                                                                   |
| `start.command` не запускается / Mac ругается            | Правый клик → "Открыть" → подтверди в диалоге безопасности                                                                      |
| `http://localhost:3000` не открывается                   | Окно Terminal от `start.command` закрылось. Запусти заново двойным кликом.                                                      |
| В Claude desktop нет Cowork                              | Cowork в исследовательском режиме — напиши разработчику запрос на доступ                                                        |
| `gh: command not found`                                  | Не установлен GitHub CLI — скачай с cli.github.com и установи                                                                   |
| Claude отказывается отправить на ревью / ругается на git | Что-то не так с авторизацией или конфликт. **Не пытайся починить сама** — отправь разработчику скриншот того что написал Claude |
| Claude генерирует не то                                  | Скрин + 1-2 конкретных замечания. _"Карточка должна быть шире, кнопка справа"_ работает лучше чем _"сделай красивее"_           |
