## Глазами в коде

### Через Cmd+F (поиск по файлу)

Эти паттерны — красные флаги:

- `bg-[` / `text-[#` / `border-[` — хардкод цвета. Должен быть токен темы (`bg-primary`, `text-muted-foreground`).
- `text-[` / `p-[` / `rounded-[` / `gap-[` — произвольные значения. Допустимо **только** с код-комментарием объясняющим зачем (по правилу snap-to-scale).
- `text-xl`, `text-2xl font-bold` и т.п. — текст пишется напрямую вместо `<Typography>`. Должен быть `<Typography variant="h2">` и подобное.
- `style={{` — inline-стиль. Допустим только для динамических значений (`width: ${progress}%`).

### Глазами на структуру

- **Папка.** Компонент в правильном месте: `components/ui/` (только shadcn), `components/common/` (переиспользуемое на 2+ страницах), `features/<имя>/components/` (внутри фичи)?
- **Имя файла** в kebab-case (`login-form.tsx`)?
- **Экспорт именованный** (`export function LoginForm`), не default?
- **`"use client"`** — есть только если реально нужен (формы, события, хуки)? Если статичный hero — должно отсутствовать.

### Состояния

- **Для компонентов с данными** — есть `loading` (скелетон/спиннер), `empty` (текст/CTA), `error` (текст + опц. retry)?
- **Для форм** — есть `submitting` (кнопка disabled + спиннер) и `error` (под полем или общий)?

### Доступность

- У каждого `<input>` — `<Label htmlFor={id}>`?
- У кнопок-иконок — `aria-label`?
- Кликабельные элементы — `<button>` / `<a>`, не `<div onClick>`?
- Декоративные иконки — `aria-hidden`?

## Когда что-то не так

1. Опиши проблему **конкретно**: _"в LoginForm нет состояния error"_, _"аватар в TestimonialCard не круглый, должен быть круг"_, _"использован text-2xl вместо Typography variant=h2"_.
2. Открой Cowork с тем же дизайнерским чатом (или начни новый — Claude подтянет CLAUDE.md).
3. Перешли замечание, попроси переделать.
4. **Не правь руками**, даже если "быстрее". Причина: правки руками не обновляют чат и систему, и в следующий раз Claude повторит ту же ошибку.

## Когда всё ок

```bash
git add components/...           # или git add .
git commit -m "feat: add <component-name>"
git push
```

### Конвенция сообщений коммита (Conventional Commits)

- `feat: add login-form` — новый компонент / фича
- `style: adjust testimonial-card spacing` — правка стилей внутри компонента
- `fix: handle empty state in product-list` — фикс бага
- `refactor: extract avatar to common` — без изменения поведения
- `docs: update designer guide` — документация
- `chore: bump next to 16.3` — рутина (зависимости, конфиги)

Scope не обязателен, но можно добавлять `(area)`: `feat(auth): add login-form`.

## Что НЕ должно попасть в ревью

Если видишь — это **не косметика, это серьёзно**, обсуди с дизайнером отдельно:

- Своя реализация Button / Input / Card вместо использования `components/ui/*`.
- Импорты с относительными путями `../../components/` вместо `@/`.
- Хардкод цветов (любых, включая black/white/gray).
- Использование `any` в типах.
- `default export` для компонента (кроме файлов страниц Next.js).
- Дублирование уже существующего в `common/` под другим именем.

Такие вещи обычно означают, что Claude не подцепил CLAUDE.md в начале сессии. Решение — в новом чате попросить дизайнера начать с _"Я дизайнер, следуй CLAUDE.md"_, потом перегенерировать.
